Write Up

Features

For this assigment we implemented multiply graphic features that can be used on any 2D image.

Section 3.2

This part of the assigment mainly had us create basic features for photo editing. Brighten(double factor) will brighten the overall photo based on the inputted factor. We implemented this by multiplying all of the pixels by the factor. ChangeContrast(double factor) is a method that will change the contrast of a photo by getting the average luminance of a photo and puttting the average through PixelLerp. ChangeSaturation(double factor) will change the saturation of a photo by using the grayscale of the image from the average luminance. ChangeGamma(double factor) will output the gamma correction of the image by setting the color to the power of (1/factor). Crop(int x, int y, int w, int h) will crop an image getting the pixerls based on the parameters.

Section 3.3

This part of the assigment we mainly delt with implementing Quantization and Dithering. Quantize(int nbits) will output the the quantized image based on the nbits per color channel. We first get the quatum by rounding off the original pixel value and the number of bins. We then get the new color while keeping the necessary range. RandomDither(int nbits) will essential do the same thing, but with some added noise. FloydSteinbergDither(int nbits) will acquire an error diffused version of the photo.

Section 3.4

This section in our opinion, was the hardest one. We mainly focused on Blur, sharpen, and edge detection. For blur, we broke up the method into two parts: the convolve and getting the filter/mask. First we would acquire the proper mask of pixels based on the width inputted. If the width was 3, then we would have a 3x3 array to be our mask of pixels. We would then use the given convolve function to blur the image. After we did blur, sharpen was easy. We simply inputted the hardcoded sharped mask and called out convolve function. For Edge detection, we were able to detect the edge pixels by using a Sobel filter which finds the gradience of the horizontal and verticle axis to determine when a pixel is changing color greatly.

Section 3.5

For this section we focused on doing scale and shift. For Scale(int sizex, int sizey), we implemented Minify and Magnify functions for x and y. We then focused on modifying the pixels for the x location of the pixels, and then location based on the y pixels. Once it was specified whether we were using a minify ot magnify, the minify and magnify methods would use the sampling_method variable to determine which filter to use. It then would return a pixel to be stored in the new pixel location. For our scale, Scaling in the X did not work and would break out code, due to other responsibilities we were unable to finish it. Shift did something similar, except we would use the inputed paramters to shift the photo instead of scaling them. Sadly we were unable to implement it successfully in time for the assignment. It is commented out in the given section. We both sadly got called away for family emergency and this hindered us from finishing.

Section 3.6

For our nonlinear filter, we created a basic filter to try and get rid of some noise in the photo. Fun(int factor) attempts to reduce the noise of the photo by creating a median filter. We loop through all over the pixels based on a range and implement I(u,v) = median{I(u+i,v+j)|getpixel(i,j)} where i and j range from 0 to 3.
